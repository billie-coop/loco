DDAID: Shared AI Context

thoughts on persistent human-AI collaboration

I've been thinking about a problem that's been bugging me.

Every time I switch projects or come back to something after a break, I have to re-explain everything to my AI assistant. All that context we built up together just... vanishes. It's like having a conversation with someone who forgets everything overnight.

So I started working on this idea: what if documentation wasn't really for humans to read? What if it was actually a shared memory system between us and our AI collaborators?

The Problem I'm Trying to Solve

When I'm coding with AI assistance, I run into the same issues over and over:

  • I explain my project's architecture to ChatGPT. Next session? Gone.
  • I tell Claude about our auth system. Tomorrow? It has no idea.
  • I switch between projects and lose all that built-up understanding.
  • Different AI tools suggest conflicting patterns because they don't share context.

The standard answer is "write better documentation," but let's be honest - documentation is usually out of date the moment it's written. And more importantly, it's not really solving the AI context problem.

The Idea: DDAID

Documentation-Driven AI Development (DDAID) is about creating persistent shared context that both humans and AI can use. Not documentation in the traditional sense, but a living memory system.

Here's how I'm thinking about it:

Instead of: Write docs → Hope someone reads them → AI doesn't know they exist

What if: Code changes → Context updates automatically → AI remembers next time

How It Works (Or How I Think It Should)

The basic pieces:

  • Git-aware context detection - Only update what actually changed
  • Specialized agents - Different bots for architecture, APIs, security, etc.
  • Model-agnostic format - Works with any AI, not locked to one provider
  • Lives with your code - Context travels in your repo

The key insight is that this isn't about generating documentation for humans. It's about maintaining a shared understanding that persists across sessions, models, and time.

What I'm Building

I'm working on a tool called Loco that explores these ideas. It's still early, but the goal is to make human-AI collaboration more continuous - less starting from scratch, more building on what came before.

Imagine if your AI assistant remembered your project as well as you do. That's what I'm trying to build.

Why This Matters (To Me)

I switch between projects a lot. I forget things. I come back to code I wrote months ago and wonder what I was thinking. Having a system that maintains context - not just for me, but for my AI collaborators - feels like it could change how we build software.

Maybe it's not revolutionary. Maybe it's just a useful idea. But I think it's worth exploring.
